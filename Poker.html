Poker Game

New Perspectives JavaScript and Ajax, Carey, Canovatchel, 2nd Edition, Tutorial 11, Tutorial

The web page displays the poker table on which users can play video poker. The player starts the game with a predetermined amount in his or her pot, that is $500. Before each hand is dealt, the player chooses an amount to bet from the dropdown list box. The pot is reduced by the amount bet. Four betting values to choose are $25, $50, $75, and $100. The player clicks the Deal button and is dealt five cards from a poker deck.

The player can select any or all of the dealt cards to replace them with new cards randomly chosen from the deck. To retrieve new cards, the user can click Draw button. The final poker hand is evaluated and the player is rewarded according to its winning hand values (See a table of the payoffs for each of nine possible winning hands in the Web page).

After the winnings are added to the pot, a new round begins. The player can choose a new amount to bet and then click the Deal button to be dealt the next hand. The game continues until the player exhausts the money in his or her pot, as a result, it will be a game over or chooses to quit the game.

Happy playing video poker Web game.

The JavaScript files used in making this game are bdlibrary.js, poker.js, cards.js. A separate HTML page of a video poker game previously designed for my JavaScript class was embedded into this page using iframe. An HTML5 iframe helps to reduce the risk of overlapping design of my overall website and puzzle's own page design, avoid formatting errors when both CSS (Cascading Style Sheets) files for each page combined, as each looks different. The iframe allows us to insert a HTML page into a website readily with less work, time, while reducing complexity and keeping the design of each page without much changes.
